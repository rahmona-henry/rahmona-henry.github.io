Tell your non-tech friend a story about a time you:

1. Got blocked on a simple problem:

What was the problem?

In the Sprint-4 resume builder exercise one of the first tasks was to get familiar with the JQuery method .append by adding your name to #main. The exercise provided a template $("#main").append([Insert Name]); which you used and replaced Insert Name with your own name. I did this and ran the code and it would not work for me.

What problem solving techniques did you use?

I tried reading the error message provided, I tried changing the code around, I Googled similar problems online and I used Slack to ask my cohort what the problem could be.

How did you feel throughout the process?

Frustratated that I ran into a problem on step one of the task and I could not figure out what the answer was. Annoyed that I was wasting so much time on such a small obvious problem and dispapointed I could not figure out the simple answer myself.

What did you learn?

To really look at the line of code that is causing the problem and think about each piece like a variable almost that can be moved around, pulled in and out, deleted. If I remove # what will happen. If I remove .append what happens. I learned not to be afraid to take code appart and test it so you can figure out the root of the problem - in my case it was the []. THe brackets were just placeholders for teh name and need to be removed from the query once the users name was added.

2. Solved a problem in an elegant way:

What was the problem?

For the Javascript racer game in Sprint- 5, I was working on a way to restart the game once a winner was announced. I had found a method online to reload the game called .reload() which was a simple line of code but I was trying to work out where I needed to add this line of code in order to make it work. So I made a simple gameOver function that housed the .reload() method and called the function after a winner was announced.

What problem solving techniques did you use?

I searched online for examples of how this method was used in other peoples code. I tried adding the line of code into parts of main code to see if it would work. I used a pen and paper to sketch out the steps of my code and see where I would logically add this line of code to make the program work.

How did you feel throughout the process?

I felt surprised that a simple line of code could make something I persieved as complex work so eloquently and easily. It also drove home the idea of how Jquery methods work and how valuable they are when solving complex pieces of code.

What did you learn?

Not to fear trying pieces of code or methods even if they look simple and unassuming.

Explain to your non-tech friend - Using the JS Array docs as a starting point, describe what these functions do in your own words:

.map()

The .map() method will iterate or move through every item in an array or list and create a new array with values/items that have been modified by some function or process that was applied to the orginal array.

.filter()

The .filter() method is used to iterate through an array and filter out the elements/items where a given condition is not true.

.reduce()

The .reduce() method is used to iterate through an array and condense it down into one value.

Have a go at explaining how recursion compares to iteration (In your own words)

Generally speaking, recursion and iteration perform the same kinds of tasks.They both solve a complicated task one piece at a time, and combine the results. Simply put: Iteration: will use a process that keeps repeating until a task is “done”. It uses loop based repetitions that execute a certain set of instructions repeatedly, without calling the function or method. Recursion: will solve a large problem by breaking it up into smaller and smaller pieces until it can solve it and then combine the results. Recursion uses a process in which a function or method calls itself repeatedly to execute a certain set of instructions.
